Dev Center Gamma Depth App

Problem Statement

Currently we get bit_depth and gamma channels from the data provider. However, due to where the gamma "sensor" is physically located at on the drillstring (BHA → bottom hole assembly), the reading we received for gamma sensor reading has a depth offset. The true gamma sensor depth is actually "above" the bit depth. If the data we received is bit_depth → 200, gamma_ray → 10; based on the screenshot below, for that particular drillstring. The data in the new collection should be bit_depth → 200, gamma_depth → 149.76 (200 - 50.24), gamma_ray → 10

* Not-Requirement Reference: Gamma ray logging wiki page

Proposed Solution

Input Data: wits (from kafka stream) + data.drillstring (from API Data Controller)

Snapshot of the collection relationship

  1. For each wits document, identify the metadata.drillstring
  1. Use API to get data.drillstring/mongodb_id
  1. For each data.components filter by mwd
    • Logic
      if (component.family = "mwd" && component.has_gamma_sensor == true)
    gamma_offset = component.gamma_sensor_to_bit_distance
    gamma_depth = bit_depth - gamma_offset
    1. If all of the mwd component has has_gamma_sensor == false. gamma_depth == bit_depth
