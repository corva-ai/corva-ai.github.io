Authorization:

How to get authorization with an API Key.

# declaring api key...good practice to store as an environment variable if sharing docs
# place your api key here
CORVA_API_KEY='user-example-api-key-1231asf3243232'

auth = {'Authorization': os.environ[CORVA_API_KEY]}

# if you do not want to store the key in your environment variables please use this:
auth = {'Authorization': 'API ' + CORVA_API_KEY}

How to get authorization with a JWT token.

# Fill in the URL, email and password to access your JWT token.

# fill in the base url here, this is the url to corva.
API_BASE_URL = "https://api.corva.ai"
# fill in your email and password here
email = "user.name@corva.ai"
password = "myPassword"

# *************** DO NOT EDIT BELOW THIS LINE: ***************
LOGGING_LEVEL = os.getenv('LOGGING_LEVEL', 'INFO').upper()
logging.basicConfig(format='%(asctime)s - %(levelname)s - %(message)s', datefmt="%Y-%m-%dT%H:%M:%S%z", level=LOGGING_LEVEL)
logger = logging.getLogger()

user_token_uri = '{}/v1/user_token'.format(API_BASE_URL)
post_request = { "auth": { "email": email, "password": password }}

logger.info(user_token_uri)
logger.info(json.dumps(post_request))

# Get JWT Token
try:
    r = requests.post(user_token_uri,
                    headers={'Content-Type': 'application/json'},
                    data=json.dumps(post_request)
                    )
    response = r.json()
    jwt_token = response['jwt']
    print(response)
    print(jwt_token)
except:
    logger.error("Exception")
    raise

# Use the JWT Token as Authorization.
auth = {'Authorization': 'Bearer {}'.format(jwt_token)}

Get List of Wellnames:

If you do not have an asset id, this function will help you find your well by wellname.

def query_corva_wells(query):
    url = API_BASE_URL + "/v1/assets"
    page = 1
    output_list = []
    while True:
        params = {
            "order": "asc",
            "page": page,
            "per_page": 20,
            "search": query,
            "sort": "name",
            "types[]": "well"
        }
        r = requests.get(url, params=params, headers=auth)
        resp = r.json()
        if len(resp) > 0:
            output_list.extend(resp)
            page += 1
        else:
            break
    return pd.json_normalize(output_list)

query_corva_wells("My_Test_Well")

Assets in Corva:

Fetching assets in Corva

To make a call for assets in Corva, users can access this at the following endpoint: https://api.corva.ai/v1/assets

There are multiple asset types: rigs, wells, programs, and companies. In order to search for a specific kind, we'll pass the type we want as a query parameter like in the example below.

url = API_BASE_URL+"/v2/assets"
params = {
    'types': 'well',
    'per_page': 10,
    'page': 1
}
r = requests.get(url, headers=auth, params=params)

When fetching assets from Corva, we must pass in our api key to the headers and can choose to either encode our query parameters into the url itself, or pass in a dict of parameters (if using the requests library). When we make the call, we see the following output below.

url = API_BASE_URL+"/v2/assets"
params = {
    'types': 'well',
    'per_page': 10,
    'page': 1
}
r = requests.get(url, headers=auth, params=params)

print(r.json()['data'])
# example of creating a dataframe from the api call results
pd.json_normalize(r.json()['data'])

Certain api calls are paginated in corva, others are not. Regardless, there is a maximum record limit for ALL collections that no requests return more than 10,000 records at a time. Therefore, for making calls on multiple pages' worth of assets, we can create an iterator to check for existence of records. If records exist, store the results. If records do not exist, break out of the iterator, such as in the example below.

def fetch_all_assets(url):
    output = []
    page = 1
    while True:
        print("Fetching data for page {}".format(page))
        
        params = {
            'types': 'well',
            'per_page': 100,
            'page': page
        }
        
        r = requests.get(url, headers=auth, params=params)
        resp = r.json()
        print(len(resp['data']))
        if len(resp['data']) > 0:
            output.extend(resp['data'])
            page += 1
        else:
            break
    return pd.json_normalize(output)
url = API_BASE_URL+"/v2/assets"

df = fetch_all_assets(url)

df.head()

We can see in the example above, that we were able to pull data from multiple pages, and that the function broke out of the infinite loop when no data was left to pull. This is extremely helpful to locate assets in Corva. Notably, having asset_id is often the most crucial information required to be able to run queries on other collections.

Fetching Rigs in Corva

The concept of a rig in Corva is similar to that of an asset. Essentially rig is a parent of well, but both are assets. Therefore, we can find rigs in Corva in a very similar manner.

Drilling Data in Corva

Fetching Drilling Wits in Corva

There are a series of multiple aggregations of live wits data in corva. In order of processing, these are:

  • wits.raw
  • wits
  • wits.summary-1m
  • wits.summary-30m
  • wits.summary-6h

In order pull wits from Corva, we will need to supply a well asset_id in addition to any extra query parameters. Keep in mind that query limits are 10,000 records for each collection, so if crawling data for an entire well, we will need to use another iterator. This time, we'll focus on using timestamps because wits data is not paginated in the same way that the assets collection is.

# function to fetcn wits from a provided asset id
def fetch_wits(asset_id):
    url = "https://api.corva.ai/v1/data/corva/wits.summary-1m"  # base url endpoint
    timestamp = 0  # initialize timestamp 
    output = []  # create a placeholder list to store results
    # infinite loop b/c we don't know how much data is out there (dynamic)
    while True:
        # api query parameters
        param = {
            'asset_id': asset_id,
            'limit': 1000,
            'query': r"{timestamp#gt#" + str(timestamp) + "}",  # we will increment timestamp with each iteration
            'sort': r"{timestamp: 1}"  # sorting ascending is super important to crawl asset data 
        }
        print("Checking for next 1000 records greater than {}".format(pd.to_datetime(timestamp, unit='s')\
                                                                      .strftime('%y-%m-%d %H:%M:%S')))
        # crank out api call
        r = requests.get(url, headers=auth, params=param)
        resp = r.json()
        
        # if data is present, go ahead and store results....take last timestamp for query
        if len(resp) > 0:
            output.extend(resp)
            timestamp = resp[-1]['timestamp']
        # if no results (empty list), then break out of loop...all data is fetched
        else:
            break
            
    # optional...could have returned output as a json array, but normalize into a dataframe in this step
    return pd.json_normalize(output)
# grabbing a sample asset from above
asset_id = df.iloc[0]['id']
asset_id
df_wits_1m = fetch_wits(asset_id)

Fetching metrics in Corva

Corva's metrics engine is a commonly interfaced application for getting high level aggregations of performance information. It can be interfaced at the following endpoint:

  • https://api.corva.ai/v2/corva/data/metrics
    url = API_BASE_URL+"/v1/data/corva/metrics"
params = {
    'asset_id': asset_id,
    'limit': 100
}
r = requests.get(url, headers=auth, params=params)

pd.json_normalize(r.json())

We can see that data.key shows the performance metric and data.key shows the aggregation level. If we want to refine our search, we can try something like below.

  • https://api.corva.ai/v2/corva/data/metrics
    url = API_BASE_URL+"/v1/data/corva/metrics"
params = {
    'limit': 100,
    'query': r"{data.key#eq#'rop'}AND{data.type#eq#'well_section'}"
}
r = requests.get(url, headers=auth, params=params)

pd.json_normalize(r.json())

In this case, we were able to pull a list of rop metrics by well section for 100 assets. Like in the other examples, we can actually create an iterator if we'd like to pull more than 10,000 records. Additionally, we could add a query for timestamp based metrics to pull based on the last n-days, for example.

url = API_BASE_URL+"/v1/data/corva/metrics"
params = {
    'limit': 100,
    'query': r"{data.key#eq#'rop'}AND{data.type#eq#'well_section'}AND{timestamp#gte#1599724500}"
}
r = requests.get(url, headers=auth, params=params)


df_metrics = pd.json_normalize(r.json())
df_metrics.head()

Completions Data in Corva

To locate active completions assets, it's recommended to fetch the frac crews from the completions collection and then using that information to find the active assets. Due to the way Corva is designed, there is no differentiation between drilling and frac assets in Corva because they can be either or. Therefore, we cannot simply query frac assets directly from the assets call. By fetching the frac fleets and then finding the assets affiliated with those fleets, we can query completions assets in Corva.

FAQ: How do we know what the current activity of the asset is? <br> Because of the way staging works, an asset can either be wirelining, frac'ing, or in drillout for the completions platform. There is no concept of current activity located in the asset information itself. Rather, Corva's applications check the frac, drillout, and wireline collections for each asset to see which one is currently populating data. The one that is currently populating data is associated with that current activitiy. For example if an asset's frac data is currently updating, but wireline is not, the platform's applications will deduce that the application is currently frac'ing and respond accordingly.

Wireline data in Corva

The following wireline collections are available in corva listed in order from least processed/aggregated to most processed/aggregated:

  • wireline.wits.raw
  • wireline.wits

To query the wireline collection, you will need to pass in an asset_id and any other query parameters.

url = API_BASE_URL + "/v2/data/corva/wireline.wits"
params = {
