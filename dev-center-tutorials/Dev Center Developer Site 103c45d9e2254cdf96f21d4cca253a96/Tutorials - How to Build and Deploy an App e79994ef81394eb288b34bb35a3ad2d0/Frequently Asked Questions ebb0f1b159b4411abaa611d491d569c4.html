Frequently Asked Questions

Created by @Michael Cortez

Last Updated @Mar 11, 2021 11:50 AM

  • Q: How do Apps in Corva work?

    A: Apps which display useful data in the Corva platform consist of one or many Backend apps (algorithms) and a Frontend app (visualization). Apps can read from and write to Datasets, datasets can be shared between multiple apps.

  • Q: What’s kind of Apps can you build in Dev Center?

    A: Completions apps that work for Frac and Wireline operations; Categories: Analytics, Asset, Wireline, Proppant etc.

    A: Drilling apps that work for drilling operations; Categories: Analytics, Asset, Directional, Hydraulics, Torque & Drag etc

    A: Drillout apps that work for drillout operations; Categories: Drillout

  • Q: How do I delete test apps from the Dev Center?

    A: Currently the Dev Admin will delete test apps until that feature is added.

  • Q: Can I pull data from multiple wells into a single app?

    A: Yes, e.g. BHA Optimization application.

  • Q: Is there a development environment in Corva Dev Center?

    A: No. You will use your own development environment on your local machine.

  • Q: Is there a section in the tutorial that specifies which files I have to modify?

    A: Yes. Within the tutorial, the Section titled Upload and Publish App to Dev Center.

  • Q: Can I talk to two different datasets at the same time?

    A: Yes. This will be done within the CRUD operations within your code. You will make an API call to post to your own databases.

  • Q: When Posting to a dataset that I create, do I have to take care of the schema or is there a format I must follow?

    A: Here is the schema to follow: https://data.corva.ai/redoc#operation/create_api_v1_data__provider___dataset___post

  • Q: When using App Type: Real-Time Stream, is there a latency component attached to it? For example if the data is submitted at time t, we expect to see it in Corva by some t + delta? How does this work?

    A: Depending on the app it follows. Following WITS can be 250ms. Following Engineering could be 1 minute.

  • Q: I want to maintain a version of the new dataset that I am writing and replace it with a new version at a certain time interval, is that possible?

    A: API allows you to update/replace existing documents as well as delete them.

  • Q: Logging: What is the best way for me to maintain logs on the backend and where are they stored?

    A: We are adding a logger component to Python SDK that will handle this. Logs are stored on AWS Cloudwatch and can be accessed and searched via Dev Center. Both logger components and showing logs in Dev Center are currently under development.
